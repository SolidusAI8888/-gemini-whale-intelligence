# V22 Daily Diff Persistence Hotfix

本补丁只修复“每日变化对比”机制，不改变 V21 的日报结构、邮件发送方式、Cabinet OGE、政治/商界图表等核心内容。

## 修复的问题

此前报告第一屏已经有“今日新增/变化披露记录”的视觉样式，但 GitHub Actions 每次运行是临时环境，如果没有恢复上一日 `data/whale.db`，系统会把 2026-01-01 至今重新抓到的历史记录全部当成“新增”。因此会出现每天都显示一万多条新增，而不是与昨日对比。

## V22 改动

1. GitHub Actions 增加持久化数据库缓存：
   - 运行前恢复 `data/whale.db`
   - 运行后保存最新 `data/whale.db`
   - 下一天以昨日数据库为基线，只把真正新插入的 source_id 计为新增

2. 报告第一屏变化口径改为：
   - 与上次成功运行保留的数据库快照对比
   - 首次运行或没有恢复快照时，会明确提示“首次运行或未恢复历史快照；本次可能包含历史入库记录”

3. 关键行动 Top 摘要中：
   - 本次新插入披露相关的股票/标的显示黄色“新增/变化”标签
   - 非本次新增但仍在榜单的项目显示“既有”

4. 邮件发送逻辑调整：
   - `SEND_EMAIL=true` 时每天都会发送日报
   - 即使没有新增，也会发送带有“今日无新增重大变化”的日报
   - 邮件主题会显示“新增”或“无新增”

## 覆盖文件

- `.github/workflows/daily_scan.yml`
- `app/main.py`
- `app/reports/html_report.py`

## 使用说明

覆盖后手动运行一次 GitHub Actions 作为初始基线。第一次仍可能显示较多“新增”，因为它正在建立持久化数据库快照。

从第二次运行开始，如果没有新的披露 source_id，报告应显示：

> 今日无新增重大变化

如果有新的披露，报告会显示新增数量，并在关键行动摘要中高亮相关股票/标的。
